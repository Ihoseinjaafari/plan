# Life+ - سیستم جامع مدیریت زندگی (Life Management System)

## 📋 معرفی پروژه

**Life+** یک پلتفرم وب فارسی‌زبان و چندمنظوره برای مدیریت جامع زندگی است که با **PHP** ساخته شده و از **JSON** به عنوان پایگاه داده استفاده می‌کند. این پروژه به کاربران امکان می‌دهد تمام جنبه‌های زندگی خود شامل کارها، پروژه‌ها، اهداف، امور مالی، سلامت و تقویم را در یک مکان متمرکز مدیریت کنند.

---

## 🎯 ویژگی‌های اصلی

### 1. **احراز هویت و مدیریت کاربران**
- ثبت‌نام و ورود کاربران
- پروفایل کاربری با رنگ آواتار سفارشی
- مدیریت نشست‌ها (Session)
- پشتیبانی از چندین کاربر همزمان

### 2. **پلنر تسک‌ها (Planner)**
- مدیریت کارها و وظایف روزانه
- دسته‌بندی تسک‌ها (کار شخصی، اداری، یادگیری، ورزش، خرید)
- اولویت‌بندی و وضعیت تسک‌ها
- اتصال به تقویم شمسی

### 3. **مدیریت پروژه‌ها (Projects)**
- ایجاد و مدیریت پروژه‌های شخصی
- پیوند تسک‌ها به پروژه‌ها
- پیگیری پیشرفت پروژه

### 4. **لایف‌پلن (Life Plan)**
- برنامه‌ریزی بلندمدت زندگی
- گروه‌بندی اهداف و برنامه‌ها
- ویرایش و مرتب‌سازی گروه‌ها

### 5. **ویژن برد (Vision Board)**
- تابلو بصری آرزوها و اهداف
- افزودن تصاویر و یادداشت‌ها
- قابلیت جابجایی و تغییر اندازه عناصر (Drag & Drop)
- ذخیره‌سازی وضعیت تابلو

### 6. **مدیریت مالی (Finance)**
- ثبت درآمد و هزینه‌ها
- دسته‌بندی معاملات (خوراک، مسکن، حمل‌ونقل، سلامت، آموزش، تفریح)
- گزارش‌گیری و تحلیل مالی
- تراکنش‌های تکرارشونده

### 7. **سلامت و بهداشت (Health)**
- ردیابی چرخه‌های سلامتی (ویژه زنان)
- ثبت علائم و symptômes
- تقویم سلامت شخصی

### 8. **تقویم شمسی (Calendar)**
- نمایش تقویم جلالی/شمسی
- یکپارچه‌سازی با تسک‌های پلنر
- افزودن رویدادها و یادآوری‌ها
- تبدیل تاریخ شمسی به میلادی و بالعکس

### 9. **داشبورد مدیریت (Dashboard)**
- نمای کلی از تمام جنبه‌های زندگی
- خلاصه تسک‌ها، پروژه‌ها، امور مالی و سلامت
- تقویم سایدبار
- دسترسی سریع به تمام ماژول‌ها

## 🌐 معماری سیستم

### جریان احراز هویت
1. کاربر از طریق `index.php` وارد می‌شود
2. اطلاعات در `data/users.json` ذخیره می‌شود
3. Session برای کاربر ایجاد می‌شود
4. تمام ماژول‌ها بررسی session انجام می‌دهند

### ساختار داده‌ها
- هر کاربر یک `user_id` منحصر به فرد دارد
- داده‌ها بر اساس `user_id` فیلتر می‌شوند
- فایل‌های JSON در پوشه `data/` ذخیره می‌شوند
- برخی ماژول‌ها فایل‌های داده مخصوص خود را دارند

### یکپارچگی ماژول‌ها
- `includes/header.php` هدر مشترک و ناوبری را فراهم می‌کند
- تمام ماژول‌ها از یک سیستم احراز هویت مشترک استفاده می‌کنند
- داده‌ها بین ماژول‌ها به اشتراک گذاشته می‌شوند (مثلاً تسک‌ها در تقویم نمایش داده می‌شوند)

## 🚀 نحوه اجرا

### پیش‌نیازها
- سرور PHP (XAMPP, WAMP, یا Linux LAMP)
- فعال بودن extensionهای JSON و Session در PHP
- دسترسی نوشتن به پوشه `data/`

### راه‌اندازی
1. کل پروژه را در پوشه web server کپی کنید
2. مطمئن شوید پوشه `data/` قابل نوشتن است
3. مرورگر را باز کرده و به آدرس پروژه بروید
4. ثبت‌نام کنید و شروع به استفاده نمایید

## 📊 مدل داده‌ها

### کاربران (`users.json`)
```json
[
  {
    "id": "1234567890",
    "name": "نام کاربر",
    "email": "user@example.com",
    "password": "hash_رمزعبور",
    "created_at": "2024-01-01 00:00:00",
```

## 🤝 همکاری و توسعه

برای اضافه کردن ویژگی‌های جدید:
1. ساختار JSON فعلی را حفظ کنید
2. از الگوی موجود در ماژول‌های دیگر پیروی کنید
3. حتماً بررسی احراز هویت را اضافه کنید
4. داده‌ها را بر اساس `user_id` فیلتر کنید

---

## 📝 نکات مهم برای AI‌ها

اگر قصد کمک به توسعه این پروژه را دارید:

1. **همیشه Session را بررسی کنید**: تمام صفحات باید `session_start()` داشته باشند
2. **داده‌ها را بر اساس user_id فیلتر کنید**: هیچ کاربری نباید به داده‌های کاربر دیگر دسترسی داشته باشد
3. **از JSON برای ذخیره‌سازی استفاده کنید**: از پایگاه داده SQL استفاده نمی‌شود
4. **فارسی و RTL را حفظ کنید**: تمام متون باید فارسی و راست‌چین باشند
5. **از توابع موجود استفاده کنید**: توابع کمکی در فایل‌های مختلف تعریف شده‌اند
6. **تاریخ شمسی**: از توابع تبدیل تاریخ در `dashboard/index.php` الهام بگیرید
7. **هدر مشترک**: از `includes/header.php` برای ناوبری استفاده شود
